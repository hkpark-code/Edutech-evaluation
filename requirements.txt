        공교육 중심 에듀테크 기업 가치평가 웹사이트 - 요구사항 명세서
================================================================================

■ 시스템 요구사항
--------------------------------------------------------------------------------

1. 런타임 환경
   - Node.js: 18.x 이상 (권장: 20.x LTS)
   - npm: 9.x 이상 또는 yarn: 1.22.x 이상

2. 운영체제
   - Windows 10/11
   - macOS 12 (Monterey) 이상
   - Ubuntu 20.04 LTS 이상

3. 브라우저 지원
   - Chrome 90 이상 (권장)
   - Firefox 88 이상
   - Safari 14 이상
   - Edge 90 이상

4. 하드웨어 권장사양
   - CPU: 2코어 이상
   - RAM: 4GB 이상
   - 디스크: 1GB 이상 여유 공간

■ 설치 명령어
--------------------------------------------------------------------------------

# 1. Node.js 설치 확인
node --version    # v18.x 이상 확인
npm --version     # v9.x 이상 확인

# 2. 프로젝트 의존성 설치
npm install

# 3. Playwright 브라우저 설치 (테스트 시 필요)
npx playwright install

# 4. 개발 서버 실행
npm run dev

# 5. 프로덕션 빌드
npm run build

# 6. 프로덕션 서버 실행
npm start

--------------------------------------------------------------------------------

■ 기능 요구사항
--------------------------------------------------------------------------------

[FR-001] 기업 정보 입력
  - 기업명 입력 필드
  - 비즈니스 모델 복수 선택 (플랫폼형/콘텐츠형/디바이스형/서비스형/네트워크형)
  - 복수 BM 선택 시 비율 설정 (매출액/정규직원 기준, 합계 100%)
  - 기업 단계 선택 (초기/성장/성숙) - 초기 미선택 상태

[FR-002] 기본 자격 평가
  - 5개 기본 항목 + 2개 디바이스 추가 항목 Pass/Fail 선택
  - 디바이스 BM 선택 시에만 전자파적합성/유해물질규제 표시
  - 1개 이상 Fail 시 평가는 진행하되 최종 결과는 Fail 처리
  - 안내 메시지: "한 항목에서라도 Fail일 경우, 다음 단계로 평가는 진행되지만, 
                 최종 결과는 Fail로 표시됩니다."

[FR-003] 기업 역량 평가
  - 재무 안정성 평가
    * 초기 기업: "택1" 선택 방식
      - 재무 건전성: 현금흐름 안전성 또는 이자보상비율 중 1개 선택
      - 자금조달역량: 투자금액 규모 또는 투자기관 신뢰성 중 1개 선택
      - 미선택 지표의 가중치는 해당 영역 내 다른 지표에 비례 배분
    * 성장/성숙 기업: 투자 유치 실적 선택사항
      - 체크박스로 평가 포함 여부 선택
      - 제외 시 가중치는 다른 지표에 재배분
  - 운영 지속성 평가 (2개 하위 지표)
  - 기술 신뢰성 평가 (5개 하위 지표, 정보보호 인증 포함)
  - 기업 단계별 차등 가중치 적용

[FR-004] 공교육 가치 평가
  - 교육적 가치 평가 (4개 영역, 15개+ 하위 지표)
    * 국가교육과정 연계성 (6개 체크리스트)
    * 교육정책 부합도 (6개 체크리스트)
    * 현장 활용도 상호 배타적 선택 (도입기관 수 / 누적 이용자 수)
  - 기술적 가치 평가
    * 학습데이터 활용 (BM별 연동 체크리스트)
    * 학습경험 설계 (사용성/접근성/몰입형 3개 하위영역)
    * 기술 확장성 (시스템 연계/데이터 이관 2개 하위영역)
    * 핵심 스킬셋 보유 (BM별 연동 체크리스트)
    * 기술 성과물 (BM별 세부 평가 - 스케일/체크리스트 혼합)
  - 사회적 가치 평가
    * 디지털 격차 완화 (저사양 기기/오프라인/데이터 절감 3개 하위영역)

[FR-005] 평가 결과 산출 및 표시
  - 회사명, 기본자격 충족 여부, 종합 등급 및 해설 표시
  - 기업역량/공교육가치 종합 점수 및 세부 영역 점수
  - SVG 도넛 파이차트로 영역별 점수 비율 시각화
    * 원 중앙: 달성률(%)
    * 상단: 획득 점수 / 만점
  - 10등급 체계 등급 판정 (AAA~D, 4구간)
  - 세부 점수 합계 = 종합 점수 일관성 보장

[FR-006] 종합 제언 자동 생성 (영역별 구분)
  - 기본자격 Fail 시 최우선 경고 메시지 표시
  - 기업 역량 분석
    * 재무/운영/기술 영역별 강점/개선필요 항목 자동 분석
  - 공교육 가치 분석 - 공통 영역
    * 교육적/기술적/사회적 가치 공통 항목 분석
  - BM별 차별화 영역 분석
    * 선택한 BM 유형별 강점/약점 상세 분석
    * BM별 핵심 스킬셋, 학습데이터 활용, 기술 성과물 기반
  - 종합 전략 제언
    * 등급 구간(A/B/C/D)에 따른 맞춤형 전략 방향 제시

[FR-007] 사용자 편의 기능
  - 페이지 이동 시 자동 스크롤 (맨 위로)
  - 이전 버튼: 첫 단계에서는 홈으로 이동
  - 체크리스트 아코디언 UI (그룹별 접기/펼치기)
  - 2열 그리드 체크리스트 레이아웃
  - 1회용 진단 안내 문구 표시
  - 선택사항 항목 선택 취소 기능 (같은 항목 클릭 시)

[FR-008] 인쇄 기능
  - 3페이지 구성 PDF 인쇄
    * 1페이지: 기본자격 + 종합등급
    * 2페이지: 기업역량 + 공교육가치 상세
    * 3페이지: 종합제언
  - 섹션별 페이지 나눔 최적화
  - 인쇄용 헤더/푸터 자동 표시

■ 데이터 구조 요구사항
--------------------------------------------------------------------------------

[DR-001] 기업 정보 (companyInfo)
  - name: string (기업명)
  - businessModels: string[] (선택된 BM 목록)
  - businessModelRatios: object (BM별 비율)
  - ratioType: 'revenue' | 'employee' (비율 기준)
  - stage: 'early' | 'growth' | 'mature' | null (기업 단계)
  - fieldUtilizationMetric: 'institution' | 'user' | null (현장 활용도 지표)
  - selectionChoices: SelectionChoices (택1 선택 정보)
  - includeInvestmentRecord: boolean (투자 유치 실적 포함 여부)

[DR-002] 선택 정보 (SelectionChoices)
  - financialHealth: 'A' | 'B' | null (재무 건전성 택1)
  - fundingCapability: 'A' | 'B' | null (자금조달역량 택1)

[DR-003] 기본 자격 (basicQualification)
  - 5개 기본 항목 + 2개 디바이스 추가 항목: 'pass' | 'fail' | null

[DR-004] 기업 역량 (companyCapability)
  - financialStability: 7개 하위 지표 (-1 ~ maxScore)
  - operationalContinuity: 2개 하위 지표
  - technicalReliability: 5개 하위 지표

[DR-005] 공교육 가치 (publicEducationValue)
  - educationalValue: 효과성/적합성/혁신성/전문성 그룹
  - technicalValue: 공통기술/BM별역량 그룹
  - socialValue: 사회적책임/윤리준수/협력생태계 그룹
